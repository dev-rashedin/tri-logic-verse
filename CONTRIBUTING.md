# Contributing to Tri Logic Verse

Thank you for your interest in contributing to **Tri Logic Verse**!  
This guide will help you get started and make your contributions as smooth as possible.

---

## How to Contribute

1. **Fork the repository**  
   Click the "Fork" button in the top-right corner to create your own copy.

2. **Clone your fork**  
   Indented code block to avoid breaking Markdown:
   git clone https://github.com/your-username/jpg-trinity.git
   cd jpg-trinity

3. **Create a new branch**  
    git checkout -b my-feature-branch

4. **Make your changes**

   - Add example problems in `problem-solving/` folders
   - Add mini-projects in respective language folders
   - Improve documentation, syntax explanations, or "under-the-hood" content

5. **Commit your changes**  
    git add .
   git commit -m "Add description of changes"

6. **Push your branch**  
    git push origin my-feature-branch

7. **Create a Pull Request**  
   Go to your fork on GitHub and click "Compare & pull request".

---

## Guidelines

- Keep examples clear and beginner-friendly
- Follow the folder structure:  
   javascript/learn/
  javascript/problem-solving/
  python/learn/
  python/problem-solving/
  golang/learn/
  golang/problem-solving/
- Use consistent formatting and naming conventions for files and functions
- Write concise, easy-to-understand comments where necessary

---

## Reporting Issues

If you find a bug or have a suggestion, please open an issue in the repository.  
Use clear titles and describe the problem or suggestion in detail.
